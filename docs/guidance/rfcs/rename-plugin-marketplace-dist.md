# RFC: Rename `plugin-marketplace-dist/` to `dist/claude-marketplace/`

**Author:** AJ Hekman  
**Date:** 2026-04-24  
**Status:** Accepted (implemented 2026-04-29)  
**Reviewers:** Engineering Team

## Summary

Rename `plugin-marketplace-dist/` to `dist/claude-marketplace/`. 

`plugin-marketplace-dist/` is a build output generated by `scripts/build.sh`. It contains auto-wrapped plugin structures for skills that live in the `skills/` directory — simple single-file contributions that don't have full plugin boilerplate. The build scripts stage them here in proper plugin format so they can be registered in `marketplace.json` and served to the Claude Code marketplace installer. The directory is committed to git because the marketplace installer reads `marketplace.json` source entries directly and cannot regenerate the staged files itself.

This RFC proposes moving it into the existing `dist/` tree where other build outputs already live, and replacing its ambiguous name with one that clearly signals both its origin (built/generated) and its purpose (Claude marketplace staging).

## Motivation

`plugin-marketplace-dist/` has two problems:

1. **Wrong location.** The repo already has `dist/` as the home for build outputs (`dist/code/`, `dist/cowork/`). A generated staging directory sitting at the repo root, outside `dist/`, is inconsistent with that convention and easy to mistake for a hand-maintained source directory.

2. **Ambiguous name.** "plugin-marketplace-dist" doesn't clearly communicate that this is a derived artifact. Contributors have edited files here directly, only to have changes overwritten on the next build. The `dist/` prefix is a widely understood signal for "generated output — do not hand-edit."

Moving to `dist/claude-marketplace/` resolves both problems in one change.

## Proposal

Rename the staging directory:

```
plugin-marketplace-dist/   →   dist/claude-marketplace/
```

Update the single source-of-truth variable in `scripts/lib.sh`:

```bash
# Before
STAGED_DIR="$ROOT_DIR/plugin-marketplace-dist"

# After
STAGED_DIR="$ROOT_DIR/dist/claude-marketplace"
```

Update `.gitignore` to keep `dist/claude-marketplace/` committed. The directory is generated by the build scripts and must stay in git — the marketplace installer reads `marketplace.json` source entries and cannot regenerate this directory itself:

```gitignore
dist/code/
dist/cowork/
dist/.cowork-staging/
dist/.gist-staging/
# dist/claude-marketplace/ is NOT ignored — committed to git, required for marketplace resolution
```

Update `SKIP_DIRS` in `scripts/lib.sh` to exclude the new path (currently excludes `"plugin-marketplace-dist"`).

Update all references in `CLAUDE.md` and build docs.

## Why `dist/claude-marketplace/` and Not Something Else

- **`dist/` prefix** — matches the existing convention; immediately signals "generated output."
- **`claude-marketplace`** — names the destination system (Claude's plugin marketplace), not the intermediate artifact type. Clearer than `marketplace-dist` or `staged-plugins`.
- **Stays committed** — unlike `dist/code/` and `dist/cowork/`, this directory must remain in git because Claude Code's marketplace resolver reads it directly at install time. The `.gitignore` comment preserves this intent explicitly.

## Impact

| File               | Change                                                                           |
| ------------------ | -------------------------------------------------------------------------------- |
| `scripts/lib.sh`   | `STAGED_DIR` variable + `SKIP_DIRS` array                                        |
| `.gitignore`       | Replace `plugin-marketplace-dist/` comment with `dist/claude-marketplace/` entry |
| `CLAUDE.md`        | Repository layout section                                                        |
| `docs/` references | Any path references to `plugin-marketplace-dist/`                                |
| Git history        | One rename commit moves the directory                                            |

No changes to `marketplace.json`, `external-plugins.json`, or any plugin source files.

## Rollout

1. Update `STAGED_DIR` in `scripts/lib.sh`.
2. Update `SKIP_DIRS` in `scripts/lib.sh`.
3. Move the directory: `git mv plugin-marketplace-dist dist/claude-marketplace`.
4. Update `.gitignore`.
5. Run `scripts/build.sh marketplace` and confirm `marketplace.json` regenerates correctly.
6. Update `CLAUDE.md` and any docs referencing the old path.
7. Open PR, verify CI passes.

## Resolved Questions

- **Does the marketplace installer hard-code the staging path?** No — it follows `marketplace.json` `source` entries. The rename is not blocked by the installer.
- **Should `dist/claude-marketplace/` be gitignored?** No — the marketplace installer cannot regenerate this directory; it only reads what `marketplace.json` points to. The directory must stay committed, built by `scripts/build.sh`.
